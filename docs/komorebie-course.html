The Sanctuary Experience

What happens when you sit down for a deep work session?

Imagine you open Komorebie to study. You see a beautiful 3D room, an ambient soundscape, and a large timer. You start a 50-minute session on "Deep Work".

Komorebie is built differently from generic productivity apps. It’s a premium sanctuary designed to feel tactile and immersive. But how does it keep track of your time without breaking the vibe?

💡
The "Sanctuary vs. Focus" Pattern

The app splits the experience in two: Sanctuary Mode (rich 3D, leaderboards, customization) and Focus Mode (hides distractions, reduces motion, shows only the timer). This separation is built deep into the routing.

The Heartbeat: The Timer

When you start a session, a circular progress bar ticks down smoothly. Let's look at how the FocusSession component renders this without causing the entire app to stutter.

FocusSession.tsx 

const FocusSession = () => {
  const { timeLeft, duration, isActive } = useZenClock();
  
  // Calculate percentage for the SVG circle
  const progress = (timeLeft / (duration * 60)) * 100;

  return (
    <svg>
      <motion.circle
        strokeDasharray="942"
        animate={{ strokeDashoffset: 942 - (942 * progress) / 100 }}
        transition={{ duration: 1, ease: "linear" }}
      />
    </svg>
  );
};
Plain English

First, get the current time left from the global "Zen Clock" memory.

Calculate how far along we are (from 0 to 100%).

Draw a circle using SVG graphics.

Use Framer Motion to smoothly animate the circle drawing itself as the percentage drops. It moves perfectly linearly every second.

Meet the Actors

The cast of characters making up the Komorebie architecture.

Every app is like a play. The code is the script, and different technologies act out specific roles. If you know the actors, you can tell an AI exactly where to put a new feature.

UI
The Stage: React & Framer Motion

The user interface. Buttons, menus, text, and buttery smooth animations.

3D
The Set Designer: React Three Fiber (R3F)

Builds the 3D sanctuary room. It translates React code into 3D objects the browser can render.

DB
The Memory: Supabase (Postgres)

The database. It remembers who you are, how much mana you've earned, and your flashcards.

AI
The Sidekick: Edge Functions + Gemini AI

Server-side logic that securely talks to the AI to summarize your notes without exposing API keys.

The Project Structure

When you ask an AI to "add a feature," knowing which folder it belongs in makes the AI 10x smarter. Here is how Komorebie is organized.

komorebie-app/src/ The main frontend application
components/ UI pieces (buttons, timer, 3D scenes)
context/ Global memories (Auth, Clock, Soundscape)
pages/ Full screens (FocusSession, Leaderboard)
hooks/ Custom reusable logic (useHabits, useFriends)
supabase/ Backend configuration
migrations/ Instructions for creating database tables

The App Router

Komorebie decides what to show you using a Router. It acts as the air traffic controller for the app.

App.tsx 

<Routes>
  {/* Landing Page */}
  <Route path="/" element={<RootHandler />} />
  
  {/* Main App (Sanctuary Mode) */}
  <Route path="/app" element={
    <ProtectedRoute>
      <AppRoot><AppLayout /></AppRoot>
    </ProtectedRoute>
  }>
    <Route path="habits" element={<HabitsPage />} />
  </Route>
</Routes>
Plain English

Define all the possible URLs in our application.

If they visit the home URL (/), run a check to see if they are logged in or a first-time visitor.

If they visit /app, first make sure they are logged in (Protected Route). If they are, wrap the screen in the 3D App Root.

If the URL specifically is /app/habits, load the Habits page inside that secure shell.

How AI safely generates flashcards from your notes.

Komorebie features an AI sidekick that reads your notes and automatically creates flashcards. But giving a website direct access to an AI brain is dangerous—hackers could steal your AI budget. So, the app uses an Edge Function.
