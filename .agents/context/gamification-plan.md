# Komorebie Gamification Plan

## Goal
- Increase retention and satisfaction without turning Komorebie into a noisy game.
- Use progression to reinforce deep work, not to compete with it.

## Design Principle
- Reward calm consistency over frantic engagement.
- The user should feel quietly rewarded after focus, never interrupted during focus.

## Core Retention Thesis
- Users return when the product helps them:
  - start quickly
  - feel progress
  - preserve continuity
  - notice improvement
  - personalize their sanctuary
- Gamification should amplify those outcomes instead of adding another task layer.

## Mana System

### Purpose
- `Mana` is the primary earned currency.
- It converts completed focus into visible sanctuary progress.
- It should make focus feel cumulative and restorative.

### What Mana Represents
- Stored attention energy from real completed work.
- It is not a random reward and not a premium currency.

### How Mana Is Earned
- Award mana only for meaningful completion events:
  - completed focus session
  - task completed after one or more sessions
  - daily focus goal met
  - weekly consistency goal met
  - note-to-flashcard study loop completed

### Mana Formula
- Base rule:
  - `1 mana per completed focus minute`
- Completion bonus:
  - `+5 mana` for finishing a planned session without early exit
- Task closure bonus:
  - `+10 mana` when the linked task is marked done
- Consistency bonus:
  - `+5%` after 3-day streak
  - `+10%` after 7-day streak
  - cap streak multiplier at `+20%`

### Anti-Abuse Rules
- no mana for abandoned sessions under 10 minutes
- reduced payout for repeated cancel/restart behavior
- no idle background farming
- no leaderboard advantage from purchased items

## What Mana Buys

### Room Unlocks
- new ambience palettes
- subtle lighting scenes
- weather layers
- desk objects
- shelf items
- window views

### Companion Growth
- companion evolution stages
- idle behavior unlocks
- visual traits
- rare sanctuary interactions

### Ritual Rewards
- unlockable start chimes
- session-complete visual flourishes
- journal card styles
- summary themes

### What Mana Should Not Buy
- stronger productivity stats
- pay-to-win ranking boosts
- intrusive effects in focus mode
- anything that pressures grinding

## Progression Model

### Layer 1: Immediate Reward
- user completes a session
- sees:
  - minutes completed
  - mana earned
  - streak status
  - one small room or companion progress signal

### Layer 2: Weekly Momentum
- user works toward:
  - weekly focus target
  - consistency streak
  - ritual completion count
- reward:
  - larger mana burst
  - new ambience unlock
  - companion affinity gain

### Layer 3: Long-Term Sanctuary Growth
- room becomes richer over time
- companion becomes more alive and responsive
- analytics reflect mastery and rhythm

## Companion System

### Role
- The companion is a calm witness to progress, not a pet demanding attention.

### Companion Stats
- `Affinity`: earned from consistent return behavior
- `Vitality`: earned from completed focus sessions
- `Wisdom`: earned from note, review, and flashcard usage

### How Stats Are Used
- `Affinity` unlocks relationship moments and presence animations
- `Vitality` unlocks visual evolution and sanctuary energy changes
- `Wisdom` unlocks reflective insights and scholarly cosmetics

### Interaction Rules
- no hunger mechanics
- no punishment for missing days
- no pushy caretaker loop
- interaction remains optional outside focus mode

## Retention Mechanics That Fit The Product

### 1. Return-To-Room Continuity
- When users come back, the sanctuary should remember:
  - last task
  - last ambience preset
  - companion state
  - unfinished plan for today
- This increases attachment without adding UI clutter.

### 2. Gentle Streaks
- Use `soft streaks`, not punitive streaks.
- Missing a day should not hard-reset identity progress.
- Model:
  - `focus streak` for consecutive days
  - `rhythm score` for consistency across the last 7 or 14 days
- Preserve some rewards through streak breaks.

### 3. Daily Intent Ritual
- One short prompt on sanctuary entry:
  - `What are you protecting time for today?`
- User can choose one priority task or one session goal.
- Reward completing the declared intent with a modest bonus.

### 4. Weekly Reflection Loop
- Weekly recap should emphasize:
  - best focus window
  - total time protected
  - tasks completed
  - sanctuary progress unlocked
- End with one recommendation, not a dashboard dump.

### 5. Collection Without Hoarding Pressure
- Users can unlock:
  - room scenes
  - soundscapes
  - companion forms
  - desk artifacts
- Collections should feel curated and premium, not endless.

### 6. Milestone Moments
- Celebrate:
  - first 60 focused minutes
  - first 7-day rhythm
  - first completed deck
  - first 10 completed tasks
  - first room evolution
- Celebration format:
  - full-screen sanctuary moment after completion
  - no confetti-style interruption during focus

### 7. Meaningful Personalization
- Let users spend mana on identity:
  - sanctuary style
  - companion expression
  - focus completion theme
- Personalization is a strong retention driver because users invest emotionally in what they shape.

## Satisfaction Levers Beyond Mana

### Fast Wins
- one-tap resume of last flow
- one-tap restart of last duration + ambience
- smart task suggestions from previous sessions

### Competence Signals
- focus summaries that show improvement over time
- subtle “best window” and “deep-work ratio” insights
- completion language that feels mature, not childish

### Emotional Payoff
- calming room transitions after successful sessions
- companion acknowledgment on return
- sanctuary visibly becoming more refined with consistent use

### Trust And Control
- mute all social prompts
- hide progression surfaces if user wants pure utility
- reduced motion equivalents for all celebration moments

## UI Placement Rules

### Sanctuary Home
- show:
  - current mana
  - streak
  - next unlock preview
- keep them compact and atmospheric

### Focus Setup
- show:
  - potential session reward only if lightweight
  - declared intent bonus if relevant

### Active Focus Session
- show:
  - none of the broader progression UI
- allowed:
  - an optional extremely subtle progress ring or ambience shift near the end

### Session Completion
- this is the main reward surface
- show:
  - session complete
  - mana earned
  - streak impact
  - task completion bonus
  - room or companion progress

### Room Page
- main place to spend mana
- should feel like curation, not a shop wall

## Notifications And Re-Engagement

### In-App
- use only for:
  - session completion
  - new unlock
  - weekly recap

### External
- optional reminders only
- examples:
  - planned focus block reminder
  - weekly rhythm summary
  - note review reminder

### Avoid
- scarcity spam
- “come back now” pressure
- fake urgency tied to streak loss

## Social Gamification

### Allowed
- quiet leaderboard
- friend presence
- shared milestones
- optional study circles later

### Not Allowed
- public activity feed
- loud reactions
- challenge spam
- direct interruption of focus mode

## Economy Safeguards
- premium users may get exclusive cosmetics, not better mana rates
- cosmetics should remain high-quality and restrained
- free users must still experience satisfying progression
- spending mana should feel reversible only in limited cases to avoid decision anxiety

## MVP Gamification Scope
- mana earning from completed focus sessions
- room unlock store with a small curated inventory
- soft streaks
- session completion rewards
- weekly recap
- companion stage 1 progression

## Post-MVP Expansion
- richer companion behaviors
- seasonal sanctuary sets
- collaborative social milestones
- advanced mastery tracks for study and deep-work patterns

## Success Metrics
- day-1 to day-7 return rate
- weekly active users completing at least 3 sessions
- average sessions per retained user
- mana spend rate
- room customization adoption
- companion progression adoption
- opt-out rate from progression features

## Risks
- if rewards appear during focus, the system fails
- if mana buys too many things, the product becomes gamey
- if streaks are punitive, satisfaction drops after breaks
- if social rewards dominate, the sanctuary identity weakens
